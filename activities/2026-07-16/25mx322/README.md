# SDLC & Agile Workflows

## Project

**Enterprise Client Portal**

This project explains how an enterprise client portal is developed using the **Software Development Life Cycle (SDLC)** and **Agile methodology**.

---

## SDLC Phases

1. **Requirement Analysis** – Collect client requirements.
2. **Planning** – Plan tasks, budget, and timeline.
3. **Design** – Design the UI, database, and system.
4. **Development** – Build the frontend and backend.
5. **Testing** – Test the application and fix bugs.
6. **Deployment** – Release the application to users.
7. **Maintenance** – Update the system and resolve issues.

## Epic

**As an enterprise customer, I want a secure client portal so I can manage my services online.**

---

## User Stories

1. **As a customer, I want to log in securely so that my account is safe.**

2. **As a customer, I want to view my dashboard so I can check my account details.**

3. **As a customer, I want to download invoices so I can keep my records.**

4. **As a customer, I want to create support tickets so my problems can be solved.**

5. **As an administrator, I want to manage customer accounts so I can maintain the portal.**

---

## Acceptance Criteria (Secure Login)

* User enters a valid email and password.
* Invalid login shows an error message.
* Successful login opens the dashboard.
* User session expires after inactivity.

---

## Sprint Planning

**Sprint Duration:** 2 Weeks

**Sprint Goal:** Build the login feature.

**Sprint Tasks:**

* Create login page
* Connect database
* Implement authentication
* Test the feature
